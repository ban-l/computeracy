# Redis

**Redis**（Remote Dictionary Server）是一个开源的、基于**内存**运行的**键值型（Key-Value）NoSQL 数据库**。

## 1. 核心定义：它是“内存”数据库

与 MySQL 或 PostgreSQL 等将数据存储在磁盘上的传统数据库不同，Redis 的主战场在**内存**。

- **速度极快**：由于内存的读写速度远高于磁盘，Redis 能够实现极高的吞吐量和微秒级的延迟。
- **定位**：它通常不直接作为主数据库存储海量、不常访问的数据，而是作为**缓存**或**高速中转站**。

## 2. 数据类型

虽然统称为“键值型”，但 Redis 的“值”可以是多种复杂的数据类型。这让它在处理特定业务场景时非常灵活：

- **String**（字符串）：基础的文本或二进制（如图片）。
- **List**（列表）：有序集合，可用于消息队列。
- **Hash**（哈希）：类似 Java 里的 `HashMap`，适合存储对象。
- **Set**（集合）/ **ZSet**（有序集合）：支持去重及排序，常用于排行榜。
- **Bitmap / HyperLogLog**：用于极小空间内的统计（如日活用户、打卡等）。

------

## 3. 核心特性

- **单线程模型（核心点）**：Redis 的核心网络处理模块是单线程的，这避免了多线程的上下文切换和锁竞争，配合 **IO 多路复用技术**，效率反而极高。
- **持久化机制**：虽然是内存数据库，但它能通过 **RDB**（快照）或 **AOF**（追加日志）将数据写回磁盘，防止断电后数据丢失。
- **高可用与扩展**：支持**主从复制、哨兵模式（Sentinel）以及分片集群（Cluster）**，能够支撑海量数据和高并发访问。

## 4. 典型应用场景

作为一名 Java 开发者，你可能会在以下场景频繁用到它：

1. **缓存**：减轻数据库压力，提升响应速度（最常见的用法）。
2. **分布式锁**：在微服务架构中，保证跨进程的互斥访问。
3. **计数器/排行榜**：利用原子递增操作（`INCR`）处理高频统计。
4. **会话管理**：集中式存储 Session 状态。
5. **消息队列**：List和Stream。

------

> **专家视点**：理解 Redis 的关键在于“**快**”与“**简**”。它把复杂的数据操作封装在内存中，让开发者能以极低的延迟处理复杂逻辑。
