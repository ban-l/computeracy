# Redis 全局命令

Redis 的局命令（Global Commands）：**它们不针对特定的数据结构（如 String 或 List），而是作用于整个键空间或服务器实例。**

我们可以将其分为以下四大类：

------

## 1. 键（Key）管理命令

这是平时开发中最常用的部分，用于操作和查询 Key 的状态。

| **命令**               | **说明**                   | **备注**                                             |
| ---------------------- | -------------------------- | ---------------------------------------------------- |
| **`KEYS pattern`**     | 查找所有符合给定模式的 Key | **慎用！** 在生产环境（大数据量）下会阻塞服务器。    |
| **`SCAN`**             | 渐进式遍历 Key             | 替代 `KEYS` 的安全方案，不会导致长时间阻塞。         |
| **`EXISTS key`**       | 检查 Key 是否存在          | 返回 1 表示存在，0 表示不存在。                      |
| **`TYPE key`**         | 返回 Key 所存储的值的类型  | 如 `string`, `list`, `set`, `zset`, `hash` 等。      |
| **`DEL key [key...]`** | 删除一个或多个 Key         | 阻塞式删除。                                         |
| **`UNLINK key`**       | 非阻塞删除                 | 在后台线程异步删除大 Key，推荐在生产环境代替 `DEL`。 |
| **`RENAME old new`**   | 修改 Key 的名称            | 如果新 Key 已存在，会被覆盖。                        |
| **`RANDOMKEY`**        | 从数据库中随机返回一个 Key | 常用于抽奖或采样。                                   |

## 2. 生存时间（TTL）相关命令

Redis 能够自动清理过期数据，全靠这组命令。

- **`EXPIRE key seconds`**：设置过期时间（秒）。
- **`PEXPIRE key milliseconds`**：设置过期时间（毫秒）。
- **`TTL key`**：查询剩余生存时间（秒）。返回 `-1` 表示永久，`-2` 表示 Key 不存在。
- **`PTTL key`**：同上，单位为毫秒。
- **`PERSIST key`**：移除过期时间，让 Key 变为永久有效。

------

## 3. 数据库与全局控制

Redis 默认提供 16 个数据库（索引从 0 到 15），虽然在集群环境下通常只使用 DB 0，但在单机/主从模式下这些命令很常用。

- **`SELECT index`**：切换数据库。
- **`DBSIZE`**：返回当前数据库的 Key 总数。
- **`FLUSHDB`**：清空**当前**数据库的所有数据。
- **`FLUSHALL`**：清空**所有**数据库的数据（极度危险，生产环境通常会禁用）。
- **`MOVE key db`**：将当前库的 Key 移动到指定库。

------

## 4. 服务器运维与调试

监控 Redis 的“健康状况”。

- **`INFO [section]`**：**最重要的命令**。查看服务器运行状态、内存占用、客户端连接、CPU、持久化统计等。
- **`MONITOR`**：实时打印出 Redis 服务器接收到的所有命令。用于调试，但高并发下会严重消耗性能。
- **`CONFIG GET/SET`**：动态查询或修改 Redis 配置（无需重启）。
- **`CLIENT LIST`**：列出当前连接的所有客户端。
- **`SLOWLOG`**：查询慢查询日志，帮助定位性能瓶颈。
